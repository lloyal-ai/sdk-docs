A source is any system your agents can search. File system. Web. Vector database. Email inbox. JIRA. Internal wiki. Anything with content and a way to find it.

You define the tools. The harness orchestrates agents with those tools via [spawnAgents()](/learn/agents#spawnagents).

## The contract

A source is a data backend. It provides tools and chunks — not orchestration.

```typescript
import { Source } from "@lloyal-labs/lloyal-agents";
import type { Tool } from "@lloyal-labs/lloyal-agents";
import type { Operation } from "effection";

class MySource extends Source<MyContext, MyChunk> {
  // 1. Human-readable label for this source
  readonly name = "jira";

  // 2. Data access tools this source provides
  get tools(): Tool[] {
    return [this._searchTool, this._readTool];
  }

  // 3. Late-bind runtime dependencies not available at construction
  *bind(ctx: MyContext): Operation<void> {
    this._searchTool = new JiraSearchTool(this._client, ctx.reranker);
  }

  // 4. Post-use chunks for reranking
  getChunks(): MyChunk[] {
    return this._buffer.getAll();
  }
}
```

<Steps>
  <Step title="name">
    A label. Used in output to attribute which source produced what. Keep it short — `"web"`, `"corpus"`, `"jira"`.
  </Step>

  <Step title="tools">
    The data access tools this source provides. The harness passes these to `spawnAgents()` along with its own prompt and orchestration config. The source doesn't decide how agents use the tools — the harness does.
  </Step>

  <Step title="bind(ctx)">
    Called before tools are used. Wire up runtime dependencies — reranker, API clients, database connections — that weren't available at construction time. This is a generator, so you can do async setup.
  </Step>

  <Step title="getChunks()">
    Returns buffered content for post-use reranking. During agent execution, tools fetch content (pages, documents, code files). The source buffers it. After agents complete, the harness calls `getChunks()` and reranks against the original query to select the best passages for synthesis.
  </Step>
</Steps>

## How sources compose

A pipeline can use multiple sources. The harness iterates through them, using `spawnAgents()` to orchestrate agents with each source's tools:

```typescript
import { spawnAgents } from "@lloyal-labs/lloyal-agents";

const sources = [
  new CorpusSource(resources, chunks),
  new WebSource(new TavilyProvider()),
];

for (const source of sources) {
  yield* source.bind({ reranker });

  const pool = yield* spawnAgents({
    tools: source.tools,
    systemPrompt: RESEARCH_PROMPT,    // harness owns the prompt
    tasks: questions,
    terminalTool: { name: "report", tool: reportTool },
    recursive: {                       // optional self-referential recursion
      name: "research",
      extractTasks: (args) => args.questions as string[],
    },
    reportPrompt: REPORT,
    pruneOnReport: true,
  });

  findings.push(pool.agents.map(a => a.findings).filter(Boolean));
}
```

Each source is independent. It provides tools. The harness provides prompts, orchestration config, and recursion shape. The same `spawnAgents()` call works with any source — web, corpus, database, API.

<Tip>
Sources are ordered deliberately. Put high-confidence sources first (local corpus, verified databases). Web or broad API sources go last. Between sources, a bridge stage can structure early findings as context for later sources.
</Tip>

## Building your own source

Start with the question: what can you search, and what do you get back?

```typescript
class SlackSource extends Source<{ reranker: Reranker }, SlackChunk> {
  readonly name = "slack";
  private _client: SlackClient;
  private _buffer: SlackChunk[] = [];
  private _searchTool!: SlackSearchTool;
  private _threadTool!: SlackThreadTool;

  constructor(token: string) {
    super();
    this._client = new SlackClient(token);
  }

  get tools(): Tool[] {
    return [this._searchTool, this._threadTool];
  }

  *bind(ctx: { reranker: Reranker }): Operation<void> {
    this._searchTool = new SlackSearchTool(this._client, ctx.reranker);
    this._threadTool = new SlackThreadTool(this._client);
  }

  getChunks(): SlackChunk[] { return this._buffer; }
}
```

The harness orchestrates agents with the source's tools:

```typescript
yield* source.bind({ reranker });

const pool = yield* spawnAgents({
  tools: source.tools,
  systemPrompt: SLACK_RESEARCH_PROMPT,
  tasks: questions,
  terminalTool: { name: "report", tool: reportTool },
  recursive: { name: "investigate", extractTasks: (args) => args.tasks as string[] },
  reportPrompt: REPORT,
});
```

The source provides data access. The harness provides the prompt, the recursion shape, and the orchestration policy. The same source can be used in a research pipeline, a support triage pipeline, or a monitoring pipeline — with different prompts and different orchestration for each.

<Note>
The source abstraction is intentionally thin. Three members: `name`, `tools`, `bind`. No orchestration, no prompts, no policy. You subclass `Source`, implement the contract, and pass instances to your harness.
</Note>
