The `Source` base class defines the contract every data source implements. Extend it to plug any backend into an agent pipeline.

## The Source Contract

```typescript
import type { Operation } from "effection";
import type { Tool } from "@lloyal-labs/lloyal-agents";

abstract class Source<TCtx, TChunk> {
  /** Human-readable name (e.g. 'database', 'pinecone') */
  abstract readonly name: string;

  /** Data access tools this source provides */
  abstract get tools(): Tool[];

  /** Late-bind runtime deps not available at construction time */
  *bind(_ctx: TCtx): Operation<void> {}

  /** Post-use chunks for reranking. Called after agents complete. */
  getChunks(): TChunk[] { return []; }
}
```

Three extension points. Only `name` and `tools` are required. Sources provide data access — the harness orchestrates agents with those tools via `agentPool()`.

<Note>
`bind()` is called before tools are used. Use it to set up anything that depends on runtime context — reranker instances, database connections, authentication tokens.
</Note>

  <Tab title="Database (SQL)">
    ```typescript
    import { Source } from "@lloyal-labs/lloyal-agents";
    import type { Tool } from "@lloyal-labs/lloyal-agents";
    import { agentPool } from "@lloyal-labs/lloyal-agents";

    interface Row {
      table: string;
      id: string;
      text: string;
    }

    class SqlQueryTool extends Tool<{ sql: string }> {
      readonly name = "query";
      readonly description = "Run a read-only SQL query against the database.";
      readonly parameters = {
        type: "object" as const,
        properties: { sql: { type: "string", description: "SQL SELECT query" } },
        required: ["sql"],
      };

      constructor(private db: any) { super(); }

      *execute(args: { sql: string }) {
        return yield* call(() => this.db.query(args.sql));
      }
    }

    class SchemaTool extends Tool<{}> {
      readonly name = "schema";
      readonly description = "Get the database schema (tables, columns, types).";
      readonly parameters = { type: "object" as const, properties: {} };

      constructor(private db: any) { super(); }

      *execute() {
        return yield* call(() => this.db.getSchema());
      }
    }

    class DatabaseSource extends Source<SourceContext, Row> {
      readonly name = "database";
      private _results: Row[] = [];
      private _tools: Tool | null = null;

      constructor(private db: any) { super(); }

      get tools(): Tool {
        if (!this._tools) throw new Error("bind() first");
        return this._tools;
      }

      *bind(ctx: SourceContext) {
        const queryTool = new SqlQueryTool(this.db);
        const schemaTool = new SchemaTool(this.db);
        const tools = [queryTool, schemaTool, ctx.reportTool]);

        // Tools are set in the constructor — no research tool needed.
        // The harness calls agentPool() with source.tools.
      }

      getChunks(): Row[] { return this._results; }
    }
    ```

    ```typescript
      readonly name = "api_search";
      readonly description = "Search the API for relevant records.";
      readonly parameters = {

        required: ["query"],
      };

      constructor(private baseUrl: string, private headers: Record<string, string>) {
        super();
      }

      *execute(args: { query: string }) {
        return yield* call(async () => {
          const res = await fetch(
            `${this.baseUrl}/search?q=${encodeURIComponent(args.query)}`,
            { headers: this.headers },
          );
          return res.json();
        });
      }
    }

    class ApiDetailTool extends Tool<{ id: string }> {
      readonly name = "api_detail";
      readonly description = "Fetch full details for a specific record by ID.";
      readonly parameters = {
        type: "object" as const,
        properties: { id: { type: "string", description: "Record ID" } },
        required: ["id"],
      };

      constructor(private baseUrl: string, private headers: Record<string, string>) {
        super();
      }

      *execute(args: { id: string }) {
        return yield* call(async () => {
          const res = await fetch(
            `${this.baseUrl}/records/${args.id}`,
            { headers: this.headers },
          );
          return res.json();
        });
      }
    }

    class APISource extends Source<SourceContext, ApiResult> {
      readonly name = "api";
      private _results: ApiResult[] = [];
      private _tools: Tool | null = null;

      constructor(
        private baseUrl: string,
        private headers: Record<string, string> = {},

      }
    ```

## Key Patterns

Every custom source follows the same structure:

1. **Define grounding tools.** These are your source-specific search and retrieval tools. Agents use them to investigate.

2. **Buffer results.** Store retrieved content during research so `getChunks()` can return it for post-research reranking.

3. **Build a research tool.** The research tool spawns a sub-agent pool using `useAgentPool` or `agentPool` -- the same primitives the built-in sources use. Pass tools directly to `agentPool`.

4. **Late-bind in `bind()`.** Set up anything that depends on runtime context: reranker, report tool, pressure thresholds. Construction happens early; binding happens when the pipeline is ready.

<Tip>
The `tools` getter is optional but valuable. It exposes your source's tools for independent verification during the synthesis phase. The synthesizer can use them to spot-check claims without re-running the full research pipeline.
</Tip>

## Plugging Into the Pipeline

Custom sources work identically to built-in ones:

```typescript
const sources = [
  new DatabaseSource(db),
  new WebSource(new TavilyProvider()),
];

// Pass to the harness -- sources run sequentially,
// each getting the full KV budget
const opts = { sources, session, reranker, /* ... */ };
yield* handleQuery(query, opts);
```

Sources run in order. After each source completes, its branches are pruned and KV is freed for the next. A [bridge](/reference/rig/bridges) structures discoveries between sources so the next source targets gaps rather than re-covering established ground.
