Tomorrow morning, an email lands in your inbox. Subject: *Daily Digest — 2026-04-30*. Body: a one-line headline summarizing the day, three or four stories with sources, a one-line "watching" closer.

You wrote ~100 lines of TypeScript on Monday. Now it runs on a cron, on a local model, costing $0 in inference. This page is how you build it.

## Prerequisites

Four things you'll need on hand:

1. **Inference model** — any instruction-tuned GGUF with native tool calling. [Qwen3-4B-Instruct](https://huggingface.co/unsloth/Qwen3-4B-Instruct-2507-GGUF) (~2.5 GB at Q4_K_M) is the reference choice.
2. **Reranker** — a small cross-encoder (`Qwen3-Reranker-0.6B`, ~400 MB at Q4_K_M). Required for any RIG-based harness — it powers chunk reranking on `fetch_page`, semantic echo guards on delegation, entailment scoring against the original query, and explore/exploit decisions at tool-dispatch time. Treat it as standard RIG infrastructure, like the inference model itself.
3. **Tavily API key** — free tier with 1k searches/month at [tavily.com](https://tavily.com).
4. **Mailgun API key + domain** — free sandbox at [mailgun.com](https://www.mailgun.com); the sandbox domain works for emailing yourself.

Set the env vars:

```bash
export TAVILY_API_KEY="tvly-..."
export MAILGUN_API_KEY="..."
export MAILGUN_DOMAIN="sandbox-xxxxx.mailgun.org"
export DIGEST_TO="you@example.com"
```

## The harness

```typescript daily-digest.ts
import { main, call } from "effection";
import { createContext } from "@lloyal-labs/lloyal.node";
import {
  initAgents, agentPool, chain, Tool,
} from "@lloyal-labs/lloyal-agents";
import type { JsonSchema, Operation } from "@lloyal-labs/lloyal-agents";
import { WebSource, TavilyProvider, reportTool } from "@lloyal-labs/rig";

// ── Mailgun delivery tool — wraps the v3 Messages API ──────────
class MailgunTool extends Tool<{ subject: string; html: string; to: string }> {
  readonly name = "send_email";
  readonly description = "Send an HTML email via Mailgun. Use this once the digest is fully drafted.";
  readonly parameters: JsonSchema = {
    type: "object",
    properties: {
      subject: { type: "string", description: "Email subject line" },
      html:    { type: "string", description: "Email body, HTML" },
      to:      { type: "string", description: "Recipient address" },
    },
    required: ["subject", "html", "to"],
  };

  *execute(args: { subject: string; html: string; to: string }): Operation<unknown> {
    const apiKey = process.env.MAILGUN_API_KEY!;
    const domain = process.env.MAILGUN_DOMAIN!;
    const auth = Buffer.from(`api:${apiKey}`).toString("base64");
    const result: unknown = yield* call(() =>
      fetch(`https://api.mailgun.net/v3/${domain}/messages`, {
        method: "POST",
        headers: {
          "Authorization": `Basic ${auth}`,
          "Content-Type": "application/x-www-form-urlencoded",
        },
        body: new URLSearchParams({
          from: `Daily Digest <digest@${domain}>`,
          to: args.to,
          subject: args.subject,
          html: args.html,
        }).toString(),
      }).then(r => r.json()),
    );
    return result;  // { id, message: 'Queued. Thank you.' } on success
  }
}

const TOPIC = process.argv[3] || "AI safety news this week";
const RECIPIENT = process.env.DIGEST_TO || "me@example.com";
const TODAY = new Date().toISOString().slice(0, 10);

const RESEARCH_PROMPT = `You are a researcher. Use web_search to surface 3-5 stories on the topic, then fetch_page on the highest-signal results to extract details. Anchor queries on the current year. When done, call report() with a concise list — each story: title, source URL, 2-sentence summary, key claim.`;

const DIGEST_PROMPT = `Read the research turn above. Draft a daily digest in HTML:
  <h2>headline summarizing the day in one line</h2>
  <ol> for each story: <li><b>title</b> (<a href="url">source</a>) — 2-3 sentence summary, then bold key claim.</li>
  <p><i>Watching:</i> one-line forward look.</p>
Once the digest HTML is ready, call send_email() with subject "Daily Digest — ${TODAY}", html=<your draft>, to="${RECIPIENT}". Then call report() with "sent: <message_id>".`;

await main(function* () {
  // 1. Load model + reranker
  const ctx = yield* call(() => createContext({
    modelPath: process.argv[2] || "model.gguf",
    nCtx: 32768,
    nSeqMax: 4,
    typeK: "q4_0",
    typeV: "q4_0",
  }));
  const reranker = yield* call(() => createContext({
    modelPath: process.env.RERANKER_PATH || "reranker.gguf",
    nCtx: 4096, nSeqMax: 1, typeK: "q4_0", typeV: "q4_0",
    pooling: "mean",
  }));

  yield* initAgents(ctx);

  // 2. Wire sources and tools
  const web = new WebSource(new TavilyProvider(process.env.TAVILY_API_KEY!));
  yield* web.bind({ reranker });

  // 3. Two-step chain — research, then synth-and-deliver
  const pool = yield* agentPool({
    orchestrate: chain(
      [
        { content: `Topic: ${TOPIC}`,    systemPrompt: RESEARCH_PROMPT },
        { content: `Compose and send.`,  systemPrompt: DIGEST_PROMPT },
      ],
      (spec, i) => ({
        task: { ...spec, seed: 100 + i },
        userContent: i === 0
          ? `Step 1: research "${TOPIC}"`
          : `Step 2: compose digest, send via Mailgun`,
      }),
    ),
    tools: [...web.tools, new MailgunTool(), reportTool],
    terminalTool: "report",
    maxTurns: 10,
  });

  pool.agents.forEach((a, i) => {
    console.log(`\n=== step ${i + 1} ===`);
    console.log(a.result || "(no result)");
  });
});
```

## What just happened

When step 2's agent starts decoding, this is what its KV holds:

```
┌──────────────────────────────────────────────────┐
│  System prompt + tool schemas                    │  ← prefilled at root
├──────────────────────────────────────────────────┤
│  user:   "Step 1: research 'AI safety…' "        │  ← extendRoot turn
│  asst:   "Found 5 stories. (1) <title>… (2) …"   │  ← step 1's REAL report
│          (URLs, summaries, claims — all of it)   │     in attention
├──────────────────────────────────────────────────┤
│  user:   "Step 2: compose digest, send via …"    │  ← step 2's task
└──────────────────────────────────────────────────┘
                ↑
        step 2's agent forks here
```

Step 2's agent **attends over** step 1's report on every token it generates. When it drafts the digest, the URLs, summaries, and key claims are already in its attention window — it doesn't need them re-fed in a prompt, doesn't need to be "told" what step 1 found, doesn't need re-tokenization. It composes from the source material directly, then calls `send_email` with the result, then `report`.

You didn't write code to:
- Extract step 1's findings into a string and stuff them into step 2's prompt
- Re-tokenize the research (cost compounds in long chains and hurts wall time)
- Manage the sequencing between steps
- Clean up step 1's branch when step 2 starts
- Wire up the model's grammar to disambiguate `web_search` from `fetch_page` from `send_email` from `report`

The framework did all of it. **The chain orchestrator's `userContent` is what becomes the curated turn on the spine; `extendRoot` writes it; the tick loop's Phase 0 prefills it onto the spine root before step 2 forks.** See [Continuous Context](/reference/continuous-context-spine) for the full mechanics.

## Make it yours

The whole orchestration is one parameter — `orchestrate`. Drop in any of these to reshape the harness:

```typescript
// Independent topics — fan out, no spine extension. Each agent runs on the same GPU.
orchestrate: parallel(topics.map(t => ({ content: t, systemPrompt: RESEARCH_PROMPT, seed })))

// A landscape pass that informs N parallel domain agents
orchestrate: fanout(landscapeStep, domainSteps)

// Arbitrary dependency graph — research two topics in parallel, compare, synth
orchestrate: dag([
  { id: 'research_A', task: { content: topicA, systemPrompt: RESEARCH_PROMPT } },
  { id: 'research_B', task: { content: topicB, systemPrompt: RESEARCH_PROMPT } },
  { id: 'compare',    dependsOn: ['research_A', 'research_B'],
    task: { content: 'Compare A vs B', systemPrompt: COMPARE_PROMPT } },
  { id: 'synth',      dependsOn: ['compare'],
    task: { content: 'Synthesize',     systemPrompt: SYNTH_PROMPT } },
])
```

The `dag` shape is what `examples/compare/` uses — a real harness that compares two subjects across multiple axes in 250 LOC. Read it after this quick-start.

## What you didn't have to write

- A message-array threading scheme — agents read prior steps from KV directly
- A token-budget tracker — the policy enforces soft and hard limits per agent
- A retry-on-error path — tool errors prefill back as `{ error: "..." }` and the agent adapts
- A teardown handler — Effection scope exits prune branches automatically
- Any explicit thread/worker code — one fiber per agent, cooperatively scheduled, all on one `llama_context`
