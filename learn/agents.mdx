Three agents. One GPU. Shared prefix.

```
Shared root (system prompt + tool schemas): decoded once
  ├── Agent 0: fork → investigate question A
  ├── Agent 1: fork → investigate question B
  └── Agent 2: fork → investigate question C
```

Every agent inherits the root's KV cache at zero cost — an O(1) metadata tag, not a copy. Each agent then generates independently, calls tools, and reports findings. The pool manages the entire lifecycle.

## What is an Agent?

An agent is a **branch with intent**. A Branch is a forkable KV cache sequence — it stores every token the model has seen and generated. An Agent adds: a task to accomplish, a policy for how to accomplish it, and a record of what it has done.

The branch is the ground truth. The agent is the interpretation layer that gives meaning to what's in the branch and makes decisions about what to do next.

```typescript
import { Agent } from "@lloyal-labs/lloyal-agents";

// createAgent and createAgentPool return Agents.
// After completion, each agent exposes:
agent.result          // what the agent discovered (or null)
agent.rawOutput       // raw model output from the final turn
agent.toolCallCount   // how many tools it called
agent.tokenCount      // how many tokens it generated
agent.nestedResults   // results from recursive sub-agents
agent.ppl             // model-level perplexity at completion
```

### Status model

Every agent transitions through a lifecycle:

```
idle → active → awaiting_tool → active → ... → idle → disposed
```

- **idle** — created but not yet generating, OR finished but branch still alive (extraction window)
- **active** — generating tokens between stop boundaries
- **awaiting_tool** — tool call parsed, waiting for result in SETTLE
- **disposed** — branch pruned, agent no longer usable

The pool enforces valid transitions. An agent cannot go backward or skip states.

### Findings

Agents produce results through a single write path with provenance tracking:

- `report_tool` — agent voluntarily called `report()`
- `free_text` — agent emitted prose without a tool call
- `scratchpad` — extracted post-idle via grammar-constrained fork
- `tool_error` — tool threw, error captured as result

### Tool history

Each agent records metadata for every tool invocation — tool name, arguments, result token count, context available after settlement. The [Agent Policy](#agent-policy) reads this history for informed lifecycle decisions.

### Parent lineage

When a tool spawns sub-agents (recursive tools like `research` or `web_research`), each sub-agent records which agent called the tool that created it. `walkAncestors()` traverses the chain — self → caller → caller's caller — collecting data from each ancestor. Tool guards use this for cross-level deduplication.

## Single agent: `createAgent`

The simplest entry point. One agent, one task, automatic lifecycle:

```typescript
import { createAgent } from "@lloyal-labs/lloyal-agents";

const agent = yield* createAgent({
  systemPrompt: "You are a research assistant.",
  task: "Find information about voice agent architectures.",
  tools: [searchTool, fetchTool, reportTool],
  terminalTool: "report",
});

console.log(agent.result);
```

`createAgent` handles everything internally: builds the toolkit, creates a shared root, forks an agent branch, runs the tick loop, and prunes all branches when done. You declare what you want — the SDK orchestrates it.

### Structured output with `schema`

For single agents that should produce structured JSON instead of calling tools:

```typescript
const agent = yield* createAgent({
  systemPrompt: "You analyze research queries. Output JSON only.",
  task: `Analyze: "${query}"`,
  schema: {
    type: "object",
    properties: {
      questions: { type: "array", items: { type: "object", properties: { text: { type: "string" }, intent: { type: "string" } }, required: ["text", "intent"] } }
    },
    required: ["questions"],
  },
});
const plan = JSON.parse(agent.rawOutput);
```

The `schema` is compiled to a GBNF grammar and set as an eager constraint — every token is grammar-valid. No retry logic, no post-hoc validation.

### Warm path via `session`

When a Session has a trunk (previous turns committed), pass it to continue from the prior conversation state:

```typescript
const agent = yield* createAgent({
  systemPrompt: EVAL_PROMPT,
  task: evalQuestion,
  schema: evalSchema,
  session,  // agent forks from session.trunk — inherits prior KV
});
```

## Parallel agents: `createAgentPool`

When a task decomposes into sub-questions, run them in parallel:

```typescript
import { createAgentPool } from "@lloyal-labs/lloyal-agents";

const pool = yield* createAgentPool({
  tasks: questions.map(q => ({ content: q })),
  tools: [searchTool, fetchTool, reportTool],
  systemPrompt: RESEARCH_PROMPT,
  terminalTool: "report",
  maxTurns: 20,
  policy: researchPolicy,
  pruneOnReport: true,
});

// pool.agents[0].result, pool.agents[1].result, ...
```

`createAgentPool` composes `withSharedRoot` + toolkit construction + the tick loop internally. Agents fork from a shared root, advance in the same GPU batch, and report independently.

### Per-task system prompts

Each task can carry its own system prompt while sharing the same KV prefix:

```typescript
const pool = yield* createAgentPool({
  tasks: questions.map((q, i) => ({
    content: q,
    systemPrompt: renderPromptForAgent(q, i, questions),
  })),
  systemPrompt: DEFAULT_PROMPT,  // shared root prefix + fallback
  tools: [searchTool, reportTool],
  terminalTool: "report",
});
```

Per-task prompts are formatted independently by each agent's `setupAgent` call — the shared root provides the KV prefix, individual agents diverge after fork.

### Recursive agents

When `recursive` is enabled, agents can spawn sub-pools via a delegation tool:

```typescript
const pool = yield* createAgentPool({
  tasks: questions.map(q => ({ content: q })),
  tools: [searchTool, fetchTool, reportTool],
  systemPrompt: RESEARCH_PROMPT,
  terminalTool: "report",
  recursive: {
    name: "web_research",
    description: "Spawn parallel sub-agents to investigate sub-questions.",
    argsSchema: { type: "object", properties: { questions: { type: "array", items: { type: "string" } } }, required: ["questions"] },
    extractTasks: (args) => args.questions as string[],
  },
  scorer,   // propagated to all depths via ToolContext
});
```

The delegation tool calls `createAgentPool` recursively — sub-agents inherit the calling agent's full KV state via Continuous Context. Recursion depth is bounded by KV pressure, not by a hardcoded limit.

### Recovery

When agents are killed by KV pressure before calling `report()`, the policy extracts partial findings. Configure `recovery` on the policy:

```typescript
const researchPolicy = new DefaultAgentPolicy({
  recovery: {
    prompt: { system: "Call report with your findings.", user: "Report." },
    minTokens: 100,     // skip extraction below this (default 100)
    minToolCalls: 2,     // skip extraction below this (default 2)
  },
});
```

Recovery prefills a nudge prompt into the agent's existing branch and forces an eager `{ result: string }` grammar. The agent's full conversation context is still in KV — the grammar forces structured extraction from it.

### `pruneOnReport`

When `true`, agents are pruned immediately after reporting. This frees KV mid-pool, giving remaining agents more headroom. Without it, all branches stay alive until the pool completes.

## `useAgent` and `useAgentPool`

`createAgent` and `createAgentPool` are the scoped wrappers — branches are pruned when they return. When you need branches to stay alive after the pool completes (for forking, verification, or follow-up):

```typescript
import { useAgent, useAgentPool } from "@lloyal-labs/lloyal-agents";

// useAgent — resource, branch stays alive
const agent = yield* useAgent({
  systemPrompt: "...",
  task: "...",
  tools: [...],
});
// agent.branch is alive — fork from it for verification
const verifyBranch = agent.branch.forkSync();
// Branch pruned when the enclosing scope exits
```

`useAgentPool` returns a `Subscription<AgentEvent, AgentPoolResult>` — events stream during execution, result is the close value:

```typescript
const sub = yield* useAgentPool({ tasks, tools, ... });
let next = yield* sub.next();
while (!next.done) {
  handleEvent(next.value);  // AgentEvent
  next = yield* sub.next();
}
const pool = next.value;     // AgentPoolResult
```

Most code never uses these directly — `createAgent` and `createAgentPool` handle the Subscription drain and scope cleanup internally.

## Agent Policy

The pool consults an **AgentPolicy** at every decision boundary — when an agent hits a stop token, when deciding whether to kill under pressure, and when checking tool guards before dispatch.

```typescript
interface AgentPolicy {
  onProduced(agent, parsed, pressure, config): ProduceAction;
  shouldExit?(agent, pressure): boolean;    // trailing stop kill gate
  shouldExplore?(agent, pressure): boolean; // optional — defaults to true
  onRecovery?(agent): RecoveryAction;       // scratchpad extraction
  resetTick?(): void;                       // per-tick flag reset
}
```

`DefaultAgentPolicy` handles all the default behavior:

- **Terminal tool interception** — when the agent calls `report()`, extract result and mark idle
- **Budget enforcement** — deny non-terminal tool calls when over budget, nudge the agent to report
- **Confabulation prevention** — reject premature reports from agents with fewer than 2 tool calls
- **Tool guard checking** — run declarative guards before dispatching any tool call
- **Explore/exploit switching** — decide per-DISPATCH whether content-boundary tools should explore (agent-local scoring) or exploit (dual scoring against original query)

Customize thresholds and guards via the constructor:

```typescript
import { DefaultAgentPolicy } from "@lloyal-labs/lloyal-agents";

const policy = new DefaultAgentPolicy({
  minToolCallsBeforeReport: 3,
  extraGuards: [myCustomGuard],
  exploreThreshold: 40,
  budget: { context: { softLimit: 2048 }, time: { softLimit: 600_000 } },
  recovery: { prompt: REPORT_PROMPT },
});

const pool = yield* createAgentPool({
  tasks, tools, systemPrompt,
  policy,
});
```

The policy sees the agent's full state: status, tool history, result, nudge count, pressure snapshot. It returns an action — `tool_call`, `report`, `nudge`, or `idle` — and the pool executes it.

### Tool Guards

Guards are declarative rejection rules checked before tool dispatch:

```typescript
interface ToolGuard {
  tools: string[];
  reject: (args, lineageHistory, agent) => boolean;
  message: string;
}
```

Default guards:

1. **Fetch dedup** — rejects `fetch_page` if the same URL was already fetched
2. **Search dedup** — rejects `web_search` if the same query (case-insensitive) was already searched
3. **Research-before-delegation** — rejects `web_research` / `research` unless the agent has done at least one search AND one fetch itself

### Entailment scoring

When agents recurse, their local hypotheses can drift from the original query. The reranker (a 0.6B cross-encoder with its own context, zero inference KV cost) acts as a real-time entailment oracle:

| Method | Scores against | Purpose |
|--------|---------------|---------|
| `scoreEntailmentBatch(texts)` | Original query | Filter drifted delegation questions |
| `scoreRelevanceBatch(texts, localQuery)` | min(tool query, original query) | Tighten focus in exploit mode |
| `scoreSimilarityBatch(reference, texts)` | Arbitrary reference | Echo detection — reject paraphrased delegations |

The scorer propagates through the entire recursion tree — every depth scores against the same original query.

## Context pressure

The pool monitors KV cache usage every tick and enforces two thresholds:

- **Soft limit** (default 1024 tokens) — headroom reserved for downstream work. When exhausted, non-terminal tool calls are rejected.
- **Hard limit** (default 128 tokens) — crash-prevention floor. Agents are killed before `produceSync()`.

Pressure management is automatic. You don't check headroom in your tools. The pool drops agents gracefully when the context fills — agents that reported results keep them, agents that didn't get recovered via `policy.onRecovery()`.

## Prefix sharing

The shared root is the key efficiency mechanism. System prompt + tool schemas are decoded into KV once. Every agent fork inherits them.

```
Without sharing: 3 agents x 900 tokens = 2,700 GPU decode ops
With sharing:    900 shared + 3 x 30 unique = 990 GPU decode ops
Savings:         64% fewer decode operations
```

When a recursive tool spawns sub-agents, `createAgentPool` forks from the calling agent's branch instead of creating a new root. Sub-agents inherit every tool call, every tool result, every token the parent generated — **2 tokens instead of ~760** for the prefill.

## Nested results

When agents delegate to sub-agents via recursive tools, inner results bubble up through `nestedResults`:

```typescript
const pool = yield* createAgentPool({
  tasks, tools, systemPrompt,
  recursive: { name: "web_research", ... },
});

// Primary findings — what each agent reported
pool.agents[0].result

// Nested results — what inner sub-agents found
pool.agents[0].nestedResults  // ["Sub-agent found X", "Sub-agent found Y"]
```

Any tool that returns `{ results: string[] }` or `{ nestedResults: string[] }` automatically participates — no tool-name hardcoding.

## Events

The pool emits events through the `Events` channel for every lifecycle transition:

- `agent:spawn` — agent created and prefilled
- `agent:produce` — token generated (with optional entropy/surprisal)
- `agent:tool_call` — tool invoked
- `agent:tool_result` — tool result received (with `contextAvailablePercent`)
- `agent:tool_progress` — progressive tool feedback (e.g. reranker scoring)
- `agent:report` — agent submitted result via terminal tool
- `agent:done` — agent completed
- `agent:tick` — batch decode committed (with KV pressure snapshot)

Subscribe in a spawned consumer to build TUI, logging, or monitoring:

```typescript
yield* spawn(function*() {
  for (const ev of yield* each(events)) {
    if (ev.type === "agent:report") {
      console.log(`Agent ${ev.agentId}: ${ev.result}`);
    }
    yield* each.next();
  }
});
```
