You'll create an agent that uses tools, reasons about results, and produces findings — all running locally on your machine.

## Prerequisites

Two files:

- **Inference model** (GGUF, tool calling support): [Qwen3-4B-Instruct](https://huggingface.co/unsloth/Qwen3-4B-Instruct-2507-GGUF) (~2.5 GB at Q4_K_M)
- **Reranker model** (GGUF, for semantic search): [Qwen3-Reranker-0.6B](https://huggingface.co/QuantFactory/Qwen3-Reranker-0.6B-GGUF) (~400 MB at Q4_K_M)

<Tip>
Any instruction-tuned GGUF model with native tool support works — Qwen3, Llama 3.x, Mistral, and more. The framework detects the model's chat template and generates grammar constraints automatically.
</Tip>

## Install

```bash
npm i @lloyal-labs/lloyal-agents @lloyal-labs/lloyal.node effection
```

## Create two simple tools

Before connecting to databases or APIs, start with the basics. Here are two tools any agent can use:

```typescript tools.ts
import { Tool } from "@lloyal-labs/lloyal-agents";
import type { JsonSchema, Operation } from "@lloyal-labs/lloyal-agents";

class CalculatorTool extends Tool<{ expression: string }> {
  readonly name = "calculate";
  readonly description = "Evaluate a math expression and return the result.";
  readonly parameters: JsonSchema = {
    type: "object",
    properties: {
      expression: { type: "string", description: "Math expression to evaluate" },
    },
    required: ["expression"],
  };

  *execute(args: { expression: string }): Operation<unknown> {
    try {
      const result = Function(`"use strict"; return (${args.expression})`)();
      return { result: Number(result) };
    } catch {
      return { error: "Invalid expression" };
    }
  }
}

class TimeTool extends Tool<Record<string, never>> {
  readonly name = "current_time";
  readonly description = "Get the current date and time.";
  readonly parameters: JsonSchema = { type: "object", properties: {} };

  *execute(): Operation<unknown> {
    return { time: new Date().toISOString() };
  }
}

export { CalculatorTool, TimeTool };
```

<Note>
A Tool is a generator function that returns an Operation. It can do anything — call APIs, query databases, read files, run shell commands, even spawn more agents. These two are deliberately simple.
</Note>

## Run an agent

```typescript agent.ts
import { main, call } from "effection";
import { createContext } from "@lloyal-labs/lloyal.node";
import { initAgents, agent } from "@lloyal-labs/lloyal-agents";
import { CalculatorTool, TimeTool } from "./tools";

const MODEL = process.argv[2] || "model.gguf";

await main(function* () {
  // 1. Load model
  const ctx = yield* call(() =>
    createContext({ modelPath: MODEL, nCtx: 8192, nSeqMax: 4, typeK: "q4_0", typeV: "q4_0" }),
  );

  // 2. Initialize runtime
  yield* initAgents(ctx);

  // 3. Run agent
  const result = yield* agent({
    systemPrompt: "You are a helpful assistant with access to a calculator and clock. Use your tools to answer precisely.",
    task: "What is 7 * 13 + the current hour of the day?",
    tools: [new CalculatorTool(), new TimeTool()],
  });

  console.log(result.result ?? result.rawOutput ?? "No output");
});
```

## Run it

```bash
npx tsx agent.ts ./models/Qwen3-4B-Q4_K_M.gguf
```

The agent will:
1. Call `calculate` with `7 * 13` → get `91`
2. Call `current_time` → get the current hour
3. Add them together and respond

All inference runs locally. No API calls leave your machine.

## What just happened

<Steps>
  <Step title="Model loaded into GPU memory">
    `createContext` loads the GGUF file, allocates KV cache, and prepares the inference backend.
  </Step>
  <Step title="Runtime initialized">
    `initAgents` sets up the inference session, branch store, and event infrastructure. This is the scope in which all agents run.
  </Step>
  <Step title="Agent created with tools">
    `agent` handled everything: serialized tool schemas into the system prompt, created a shared KV prefix, forked an agent branch, and ran the tick loop. Each tool result was prefilled into the agent's KV cache before the next generation step.
  </Step>
  <Step title="Cleanup happened automatically">
    When `agent` returned, all branches were pruned. No leaked GPU state. The `Agent` object holds the result — the GPU resources are already freed.
  </Step>
</Steps>

## What you get back

`agent` returns an `Agent` — a completed agent with everything it found:

```typescript
result.result        // findings from the terminal tool (if set)
result.rawOutput     // raw model output from the final turn
result.tokenCount    // total tokens generated
result.toolCallCount // number of tool calls made
```

## Scale to parallel agents

When a single agent isn't enough — multiple sub-questions, different angles of investigation — use `agentPool`:

```typescript
import { agentPool } from "@lloyal-labs/lloyal-agents";

const pool = yield* agentPool({
  tasks: [
    { content: "What is the population of Tokyo?" },
    { content: "What is the population of New York?" },
    { content: "What is the population of London?" },
  ],
  tools: [searchTool, reportTool],
  systemPrompt: "You are a research assistant. Use your tools to find accurate data.",
  terminalTool: "report",
});

// Each agent investigated independently, sharing the same GPU and KV prefix
pool.agents.forEach(a => console.log(a.result));
```

All three agents fork from the same shared root — the system prompt and tool schemas are decoded once. They advance in the same GPU batch. When one finishes, its KV is freed for the others.
